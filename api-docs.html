Complete documentation for KazJobs REST API

Base URL: http://localhost:4000/api/v1

🔐 Authentication

Protected routes require a Bearer token in the Authorization header:


Authorization: Bearer <accessToken>
Auth

Register, login and manage user sessions

POST /users/register public Create new account ▶

Request Body

{ "username": "john", // required "email": "john@email.com", // required "password": "123456", // required, min 6 "role": "seeker" // optional: seeker | employer }

Response 201

{ "message": "User created successfully.", "user": { id, username, email }, "accessToken": "eyJ..." }
POST /users/login public Login to account ▶

Request Body

{ "email": "john@email.com", // required "password": "123456" // required }

Response 200

{ "message": "User logged in successfully.", "user": { id, username, email }, "accessToken": "eyJ..." }
POST /users/logout public Logout user ▶

Request Body

{ "email": "john@email.com" // required }

Response 200

{ "message": "User logged out successfully." }
POST /users/refresh public Get new access token ▶

Request Body

{ "refreshToken": "eyJ..." // required }

Response 200

{ "accessToken": "eyJ..." }
Users

User profile and listing

GET /users/profile 🔒 any role Get own profile ▶

Headers

Authorization: Bearer <accessToken>

Response 200

{ "user": { _id, username, email, role, createdAt } }
GET /users?role=seeker|employer public Get all users ▶

Query Params

role: "seeker" | "employer" // optional filter

Response 200

{ "users": [{ _id, username, email, role }] }
Jobs

Job listings management

GET /jobs public Get all jobs ▶

Query Params

search: "node" // searches title, description, skills category: "technology" // technology | finance | healthcare | engineering | education | retail type: "full-time" // full-time | part-time | remote location: "Almaty" // any location string

Response 200

{ "jobs": [{ _id, title, description, salary, location, type, category, skills, employer, deadline }] }
GET /jobs/:id public Get single job ▶

Response 200

{ "job": { ...jobFields, employer: { username, email } } }
POST /jobs 🔒 employer Create job ▶

Request Body

{ "title": "Backend Developer", // required "description": "We need a dev...", // required "location": "Almaty", // optional "salary": 500000, // optional "type": "full-time", // full-time | part-time | remote "category": "technology", // technology | finance | healthcare | engineering | education | retail "skills": ["Node.js", "MongoDB"], // optional "deadline": "2026-06-01" // optional }
PATCH /jobs/:id 🔒 employer Update job ▶

Request Body

// any job fields you want to update { "salary": 600000, "title": "Senior Backend Developer" }
DELETE /jobs/:id 🔒 employer Delete job ▶

Response 200

{ "message": "Job deleted successfully." }
Applications

Job application management

POST /applications 🔒 seeker Apply to job ▶

Request Body

{ "job": "jobId", // required "CV": "url" // optional }
GET /applications 🔒 seeker My applications ▶

Response 200

{ "applications": [{ _id, job, applicant, status, CV, createdAt }] }
GET /applications/:id 🔒 seeker | employer Single application ▶

Response 200

{ "application": { job: { title, location, salary, employer }, applicant: { username, email }, status: "pending" | "accepted" | "rejected" } }
PATCH /applications/:id 🔒 employer Update status ▶

Request Body

{ "status": "accepted" | "rejected" | "pending" }
DELETE /applications/:id 🔒 seeker | employer Delete application ▶

Response 200

{ "message": "Application deleted successfully." }
GET /applications/job/:id 🔒 employer All apps for a job ▶

Response 200

{ "applications": [{ ...app, applicant: { username, email } }] }
